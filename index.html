Unit I – Fundamentals of HTML

Focus: Understanding what HTML is, how a web page is structured, basic tags, text formatting, quotations, comments, tag types, and block vs inline elements.
I Introduction to Web Technologies and HTML ▶

A website is built mainly using three core technologies:

  • HTML (HyperText Markup Language) – defines the structure and content of a web page (headings, paragraphs, images, forms, etc.). Think of it as the skeleton.
  • CSS (Cascading Style Sheets) – applies design: colors, fonts, spacing, layouts. Think of it as clothes and looks.
  • JavaScript – adds interactivity and behavior: validation, calculations, dynamic content. Think of it as the brain.
Web Page ├── HTML → Structure (what is on the page) ├── CSS → Style (how it looks) └── JS → Behavior (how it reacts/interacts)

HTML is a markup language, not a programming language. It uses tags (like <p>, <h1>) to mark up text and elements. Browsers read this HTML and render it visually.

S Structure of an HTML Document ▶

Every HTML document follows a standard structure. This ensures that browsers know how to interpret the content correctly. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Page</title>
  </head>

  <body>
    <h1>Hello World</h1>
    <p>This is my first HTML page.</p>
  </body>
</html>
  • <!DOCTYPE html> – tells the browser this page uses HTML5.
  • <html> – root element; everything inside it is part of the HTML document.
  • <head> – contains metadata (data about the page): title, character encoding, linked CSS/JS files.
  • <body> – contains visible content displayed in the browser window.
R Root and Metadata Elements ▶

The root element is <html>. It usually includes a lang attribute like lang="en" for English.

Metadata elements are placed inside the <head> section:

  • <title> – text shown in the browser tab.
  • <meta charset="UTF-8"> – character encoding; UTF-8 supports almost all languages.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0"> – makes page responsive (mobile friendly).
  • <link rel="stylesheet" href="styles.css"> – links an external CSS file.
  • <script src="app.js" defer></script> – links an external JavaScript file.
B Basic Tags: Headings, Paragraphs, Line Breaks, Horizontal Rules ▶

Some of the most basic HTML tags are:

  • Headings: <h1> to <h6> describe titles and subtitles.
  • Paragraph: <p> for normal text blocks.
  • Line break: <br> for breaking a line without starting a new paragraph (use rarely).
  • Horizontal rule: <hr> to draw a horizontal line, usually for separation of content. 
<h1>Main Heading</h1>
<h2>Sub Heading</h2>

<p>
  This is a paragraph. It explains something in detail.
</p>

First line<br>
Second line

<hr> <!-- horizontal line -->
T Text Formatting: strong, em, mark, small, abbr, cite, dfn, quotations, comments ▶

These tags give meaning to the text (semantic HTML), not just style:

Tag Meaning / Use Example
<strong> Strong importance (usually bold) <strong>Warning!</strong>
<em> Emphasis, stress (usually italic) <em>Very important</em>
<mark> Highlight text <mark>Key term</mark>
<small> Small print / fine notes <small>* conditions apply</small>
<abbr> Abbreviation with full form <abbr title="HyperText Markup Language">HTML</abbr>
<cite> Title of a work (book, film, etc.) <cite>The Alchemist</cite>
<dfn> Defining a term first time <dfn>API</dfn> is an interface ...

Quotations:

  • <q> – inline short quote (browser adds quotation marks).
  • <blockquote> – long block quote, usually indented. 
<p>He said, <q>Practice daily.</q></p>

<blockquote>
  “Any sufficiently advanced technology is indistinguishable from magic.”
  <cite>– Arthur C. Clarke</cite>
</blockquote>

Comments:

<!-- This will not be displayed in the browser -->

Comments help developers understand the code but are ignored by the browser.

P Paired vs Unpaired tags, Block-level vs Inline elements ▶

Paired vs Unpaired (Void) tags

  • Paired tags – <tag>...</tag>
    Example: <p>Paragraph</p>, <h1>Heading</h1>.
  • Unpaired / void tags – only opening tag, no closing tag.
    Examples: <br>, <hr>, <img>, <meta>, <input>.

Block-level vs Inline elements

Block-level elements: - Start on a new line - Occupy full available width - Can contain block and inline elements Examples: <div>, <p>, <h1>, <ul>, <table> Inline elements: - Flow inside a line of text - Only take as much width as needed - Usually cannot contain block elements Examples: <span>, <a>, <strong>, <img>

Example:

<div>
  <p>This is a block paragraph with <strong>inline</strong> text.</p>
  <span>This span is inline.</span>
</div>

Unit II – Expanding HTML Knowledge

Focus: Lists, hyperlinks, images, media, favicon, div, tables with colspan/rowspan, and forms with different controls.
L Working with Lists: Ordered, Unordered, Definition ▶

HTML supports three main types of lists:

  • Unordered list – items with bullets. <ul>
  • Ordered list – items with numbers / letters. <ol>
  • Definition list – terms and definitions. <dl> 
<!-- Unordered list -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list -->
<ol>
  <li>Step 1</li>
  <li>Step 2</li>
</ol>

<!-- Definition list -->
<dl>
  <dt>HTML</dt>
  <dd>Markup language for web pages</dd>
  <dt>CSS</dt>
  <dd>Stylesheet language for designing</dd>
</dl>
H Hyperlinks (Links between pages) ▶

Hyperlinks connect one page to another. The main tag is <a> (anchor). 

<a href="https://www.example.com">Visit Example</a>

<a href="about.html">About Us (internal link)</a>

<a href="#unit1">Go to Unit I (same page)</a>
  • href – the target URL or id.
  • target="_blank" – open in a new tab.
  • title – tooltip text.
M Inserting Images & Embedding Media ▶

Images

Use <img> to display an image. It is a void tag (no closing tag). 

<img src="photo.jpg" alt="Profile photo" width="200" height="200">
  • src: path/URL of the image file.
  • alt: alternative text for screen readers and when image fails to load.
  • width/height: image size.

Audio and Video

<audio controls>
  <source src="song.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

<video controls width="400">
  <source src="movie.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

You can also embed YouTube or other media using iframes:

<iframe width="560" height="315"
src="https://www.youtube.com/embed/VIDEO_ID"
title="YouTube video player"
frameborder="0"
allowfullscreen></iframe>
F HTML Favicon & Div / Text Formatting Review ▶

Favicon

A favicon is the small icon shown in the browser tab. It is defined in the <head> section: 

<link rel="icon" href="favicon.ico" type="image/x-icon">

<div> as a generic container

<div> is a block-level container used to group related elements for styling or layout. 

<div class="card">
  <h2>Title</h2>
  <p>Some descriptive text here.</p>
</div>

You already saw basic text formatting tags in Unit I (strong, em, etc.). They are often placed inside <div> or <p> for structure.

T Working with Tables, Colspan & Rowspan ▶

Tables display data in rows and columns. Basic tags: <table>, <tr> (table row), <th> (header cell), <td> (data cell). 

<table border="1">
  <tr>
    <th>Name</th>
    <th>Marks</th>
  </tr>
  <tr>
    <td>Amit</td>
    <td>90</td>
  </tr>
</table>

Colspan

colspan merges cells horizontally (across columns). 

<table border="1">
  <tr>
    <th colspan="2">Student Details</th>
  </tr>
  <tr>
    <td>Name</td>
    <td>Amit</td>
  </tr>
</table>

Rowspan

rowspan merges cells vertically (across rows). 

<table border="1">
  <tr>
    <th rowspan="2">Name</th>
    <th>Subject</th>
  </tr>
  <tr>
    <td>Maths</td>
  </tr>
</table>
F Working with Forms: action, GET/POST, Inputs, TextArea, Buttons, Checkboxes, Radio, Select, File ▶

Forms allow users to send data to the server. All controls are wrapped inside <form>. 

<form action="/submit" method="post">
  <label>Name:
    <input type="text" name="username">
  </label>
  <button type="submit">Send</button>
</form>
  • action – URL where the form data is sent.
  • method="get" – data is appended in URL query string (visible).
  • method="post" – data is sent in HTTP body (preferred for sensitive/large data).

Common form controls

<!-- Text input -->
<input type="text" name="fullname" placeholder="Enter your name">

<!-- Password -->
<input type="password" name="pwd">

<!-- TextArea -->
<textarea name="message" rows="4" cols="30"></textarea>

<!-- Checkbox -->
<input type="checkbox" name="subscribe" value="yes"> Subscribe to newsletter

<!-- Radio buttons (same name, different values) -->
Gender:
<input type="radio" name="gender" value="male"> Male
<input type="radio" name="gender" value="female"> Female

<!-- Select dropdown -->
<select name="country">
  <option value="in">India</option>
  <option value="us">USA</option>
</select>

<!-- File input -->
<input type="file" name="resume">

<!-- Buttons -->
<button type="submit">Submit</button>
<button type="reset">Reset</button>

Unit III – Cascading Style Sheets (CSS)

Focus: CSS rules, selectors, types of CSS (inline/internal/external), box model, background, fonts, typography, and styling tables/forms.
C Introduction to CSS & Types of CSS ▶

CSS (Cascading Style Sheets) controls how HTML elements look: colors, sizes, positions, layouts, etc.

Three ways to apply CSS

  • Inline CSS – via style attribute on the element.
    <h1 style="color: blue; font-size: 24px">Title</h1>
    Pros: Quick. Cons: Hard to maintain, not reusable.
  • Internal CSS – inside <style> in the <head>. 
    <style>
p {
  color: #333;
  line-height: 1.5;
}
</style>
  • External CSS – in a separate .css file, linked using <link>.
    <link rel="stylesheet" href="styles.css">
    Best practice for real projects.
R CSS Rules, Properties & Selectors (type, id, class, etc.) ▶

A CSS rule has three parts: selector, property, value.

selector {
  property: value;
}

Key types of selectors:

  • Type selector: p { color: red; } – applies to all <p> elements.
  • Class selector: .note { ... } – applied to elements with class="note".
  • ID selector: #main { ... } – applied to the element with id="main".
  • Descendant selectors: nav a { ... } – applies to <a> inside <nav>. 
/* Example */
h1 {
  color: #0b63d3;
}

.highlight {
  background-color: yellow;
}

#footer {
  text-align: center;
  font-size: 12px;
}
T Text Controlling & Formatting in CSS ▶

Important text-related CSS properties:

  • font-family – font face.
  • font-size – size (px, em, rem, etc.).
  • font-weight – thickness (normal, bold, 400, 700).
  • line-height – spacing between lines of text.
  • text-align – left, right, center, justify.
  • text-decoration – underline, none, etc.
  • color – text color.
p {
  font-family: "Segoe UI", Arial, sans-serif;
  font-size: 16px;
  line-height: 1.6;
  color: #333;
  text-align: justify;
}

a {
  text-decoration: none;
}
B CSS Box Model: Padding, Margin, Border, Div & Span ▶

Every element in CSS is treated as a rectangular box. The box model has:

[ Margin ] [ Border ] [ Padding ] [ Content ]
  • Content – text or image inside the box.
  • Padding – space between content and border.
  • Border – line surrounding padding and content.
  • Margin – space outside the border (gap between boxes). 
.card {
  width: 300px;
  padding: 16px;
  border: 1px solid #ccc;
  margin: 10px auto;
}

Div and span in CSS

  • <div> – block-level container, often used as a box to which CSS styles are applied.
  • <span> – inline container to style small parts within a line.
G Background Images, Fonts & Typography ▶

Background images

body {
  background-image: url("background.jpg");
  background-size: cover;        /* stretch to cover screen */
  background-repeat: no-repeat;
  background-attachment: fixed;  /* stays fixed on scroll */
}

Fonts & typography

h1 {
  font-family: "Poppins", Arial, sans-serif;
  font-size: 32px;
  font-weight: 700;
  text-align: center;
}

p {
  font-size: 16px;
  line-height: 1.7;
}
S Applying CSS on Tables & Forms (including hoverable tables) ▶

Styling tables

table {
  border-collapse: collapse;
  width: 100%;
}

th, td {
  border: 1px solid #d1d5db;
  padding: 8px;
}

tr:nth-child(even) {
  background-color: #f9fafb;
}

/* hoverable rows */
tr:hover {
  background-color: #e5edff;
}

Styling form controls

input[type="text"],
input[type="email"],
textarea,
select {
  width: 100%;
  padding: 8px;
  border-radius: 4px;
  border: 1px solid #d1d5db;
  margin-bottom: 8px;
}

button {
  padding: 8px 16px;
  border: none;
  border-radius: 4px;
  background: #2563eb;
  color: white;
}

Unit IV – JavaScript Application Development

Focus: Including JS in a page, variables (var/let/const), operators, control flow, loops, pop-ups, and basic objects.
J Including JavaScript: head, body, external file ▶

You can add JavaScript in three main ways:

  • Inline script: using <script> tag directly.
  • In the head: with <script> inside <head> (often with defer).
  • External file: <script src="app.js"></script>. 
<!-- In head (with defer) -->
<head>
  <script src="app.js" defer></script>
</head>

<!-- Or before closing body -->
<body>
  ...
  <script>
    console.log("Inline JS");
  </script>
</body>
V Variables & Operators (var, let, const) ▶

Variables

  • var – function-scoped, can be redeclared, hoisted.
  • let – block-scoped, cannot be redeclared in same block.
  • const – block-scoped, cannot be reassigned (value binding is constant). 
var a = 10;
let b = 20;
const c = 30;
// c = 40;  // Error: Assignment to constant variable

Common operators

  • Arithmetic: +, -, *, /, %
  • Comparison: ==, ===, !=, >, >=, etc.
  • Logical: && (AND), || (OR), ! (NOT)
C Control Statements: if...else, switch, break, continue ▶

if...else:

let marks = 72;

if (marks >= 90) {
  console.log("Grade A");
} else if (marks >= 60) {
  console.log("Grade B");
} else {
  console.log("Grade C");
}

switch:

let day = 2;
switch (day) {
  case 1:
    console.log("Monday");
    break;
  case 2:
    console.log("Tuesday");
    break;
  default:
    console.log("Other day");
}

break stops the loop; continue skips current iteration.

for (let i = 0; i < 5; i++) {
  if (i === 2) continue; // skip 2
  if (i === 4) break;    // stop loop
  console.log(i);
}
L Loops: while, do...while, for ▶ 
// for loop
for (let i = 0; i < 3; i++) {
  console.log("for:", i);
}

// while loop
let j = 0;
while (j < 3) {
  console.log("while:", j);
  j++;
}

// do...while loop (runs at least once)
let k = 0;
do {
  console.log("do-while:", k);
  k++;
} while (k < 3);
P Popup Boxes: alert, confirm, prompt ▶ 
alert("Simple alert message");

let ok = confirm("Do you want to continue?");
console.log("User choice:", ok); // true or false

let name = prompt("What is your name?");
console.log("Hello", name);
  • alert() – shows a message box with OK.
  • confirm() – shows OK/Cancel, returns true/false.
  • prompt() – asks for input, returns typed string (or null if cancelled).
O JavaScript Objects, Properties & Methods ▶

An object is a collection of key-value pairs. Keys are called properties, and functions inside objects are called methods. 

const person = {
  name: "Sonu",
  age: 20,
  city: "Phagwara",
  greet: function () {
    return "Hello, " + this.name;
  },
};

console.log(person.name);      // property
console.log(person.greet());   // method
